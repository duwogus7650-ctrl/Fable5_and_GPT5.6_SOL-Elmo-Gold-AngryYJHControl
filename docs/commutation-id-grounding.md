# 커뮤테이션 확립 명령 그라운딩 (P0 · A2 해소)

> 출처: fable-reader 정밀독해 2026-07-20. 주 문서 `docs/command-reference.txt`
> (MAN-G-CR Ver. 1.406, 2013), 보조 `docs/firmware-release-notes.txt`
> (우리 FW = Ch.12 `01.01.16.00 05Mar2020B01`, 줄 974~1216). Admin PDF는
> 2026-07-21 fitz로 판독 완료(맨 아래 "Admin PDF 판독 보강" 섹션) — MF=0x80 의미와
> 위저드 프리미티브(UM=3 스테퍼 phasing) 확정.

## 결론 — A2 판정: 경로 존재 (YES)

앱-주도로 커뮤테이션을 확립하는 문서화된 명령 경로가 **최소 3개** 존재한다:
① `CS`(커뮤각 직접 강제, RAM 전용), ② `CA[17]=2/3 + SC[]`(내장 스테퍼/이진 커뮤
서치 — 우리 FW는 "블록/언밸런스 내성" 개정판), ③ `CA[7]` 재기록 + `CA[17]=5`
무모션 재계산. EAS 위저드의 "대칭 이동" 전제 없이 가능한 후보 시퀀스가 있다.

**중대 모순(미해결):** CA[17]=5 문서 모델대로면 CA[7]이 같은 한 전원 사이클 후
δ가 재현되어야 하는데, 실기(원장 2026-07-15)는 CA[7] 비트동일인데 δ 0°→103° 재추첨.
**2013 CR은 이 재추첨을 설명 못 함** → 실기 질의 항목(§7).

## "유효 커뮤 = 전원 세션 RAM 상태" 근거

문서 근거로 **부분 지지**: (1) 커뮤 "발견됨"은 런타임 플래그(MO=1 서술), 플래시 저장
서술 없음. (2) CA[] 변경은 커뮤 리셋 — CA는 *입력 파라미터*, 커뮤 상태는 별도 휘발
개체. (3) CS는 명시적 RAM. (4) TW 계열은 전원/RS/SV 리셋. (5) CA[7]은 *오프셋
파라미터*지 "이번 세션 유효각"이 아님. 방법5의 유일 커뮤 입력=CA[7].
**모순**(§결론): CA[17]=5 무모션 재계산 모델 ⟺ 전원마다 δ 재추첨 실기 — CR로 판정 불가.
가능한 설명(전부 미확정): (a) 2020 FW EnDat 2.2(ID30, CR enum 밖) 유도가 CR과 다름,
(b) EAS가 CA[7] 외 RAM 보정치(TW?) 사용→전원마다 소실, (c) 엔코더 datum/멀티턴 매핑.

## EAS 위저드 추정 시퀀스 (강도 MODERATE~WEAK)

TW=위저드 내부설정 · TW[8]=1,2 Phasing 프로파일 BG · 방법5 유일 영구산출=CA[7] ·
이진서치 "방향판정 최소이동 SC[7]"이 에러문구 "Positive and Negative Movements are
Uneven"과 정합. 추정: MO=0 임시구성(TW) → MO=1 + phasing ±소진폭 모션(대칭성으로
유효판정 ← **백래시에 깨지는 지점**) → 절대센서각−전기각 오프셋 산출 → **CA[7] 기록**
(위저드 후 CA[7]=438/174 실기 관측 정합) → MO=0·복원. 확정 = Admin PDF(미독) 또는
EAS 통신 캡처 필요.

## 앱-주도 커뮤 확립 후보 (대칭 이동 전제 없음)

### 후보 A — CA[7] 재기록 + CA[17]=5 무모션 재계산 (1순위)
1. δ 실측 (기존 자산: UM=3 드래그 오라클 / K_a 서명 / breakaway 방향, HOLD-CONFIRM
   백래시 내성 재사용 — 모션은 측정 단계에만).
2. MO=0 → 3. CA[7]_new = wrap±512(CA[7]_old − δ·512/360) → 4. CA[7]=<new>
   (평문 ≤6자리·되읽기 검증, 원장 2026-07-15) → 5. MO=1→SO=1 폴(방법5 무모션 재계산)
   → 6. 서명 게이트 재측정, 불합격 시 δ 재산출·2~5 반복 → 7. 합격 시 SV(단 §모순으로
   전원 후 재현 미보장 → 매 전원 서명 게이트 유지).
   미지수: CA[7] 스테퍼-units 부호 방향 — 실기 소량 스텝으로 확정.

### 후보 B — CS 강제 (세션 스코프, 매 전원 재실행용 폴백)
- B-1(무모션, δ 기지): MO=0→UM=3→MO=1→`CS=<계산각>`(즉시·RAM).
- B-2(문서 예제 p62): `MO=0; UM=3; MO=1; TC=<전류>; PA=<각>; BG`(로터를 기지 전기각으로
  당김) → 안정 대기 → `CS=<대응각>` → MO=0.
- 미확정: UM=3→UM=5 복귀 시 강제각 유지 여부, UM=3 밖 수용. 저전류·서명 게이트 후행 필수.

### 후보 C — 내장 이진/스테퍼 서치 (δ 측정 실패 시 최후 수단)
`MO=0 → CA[17]=3(이진,minimal move) 또는 2(스테퍼) → SC[1..7] 설정 → MO=1(자동 서치,
20s 한도) → SO=1 폴 → (이진이면 SC[12] 검증)`. 우리 FW 스테퍼 커뮤 = "이동 제한/불허
블록·언밸런스 부하 내성"(문서상 감속기+백래시에 가장 강한 내성). 제약: 완료 후 CA[17]→5
되돌리면 커뮤 리셋 → (i) 세션 동안 CA[17]=3 유지 운전 또는 (ii) 서치 δ 역산→후보 A 이관.
미확정: EnDat 소켓에서 CA[17]=2/3 수용 여부(EC 36), 블록내성 신규 SC 인덱스.

## 미해결 — 실기 질의 목록
1. CA[7] 스테퍼-units 부호/기준 방향 (소량 오프셋 스텝→δ 재측정 1회).
2. 전원 사이클 δ 재추첨 기제 — 후보 A가 전원 후 재현되는지가 판별 실험.
3. CS의 UM=5 복귀 후 유지·UM=3 밖 수용 여부.
4. CA[17]=2/3의 EnDat 소켓 수용(EC 36)·블록내성 신규 SC 인덱스.
5. CA[111] 쓰기의 Hall-부재/방법5 수용·서치 강제 여부.
6. EAS 위저드 실제 시퀀스 — Admin PDF(fitz 환경) 또는 통신 캡처.

## P4 배치 권고
후보 A = 1차 경로, 후보 B(CS) = 매-전원 폴백, 후보 C = δ 측정 실패 시 최후 수단.
CR 단정 금지 — enum/인덱스 의미는 2020 FW에서 다를 수 있음(EnDat ID 9→30 전례).

## Admin PDF 판독 보강 (2026-07-21, man-g-adming Ver.1.100)

**MF=0x80(128) 확정 = "Speed tracking error"** (MF 비트필드 표, p.85):
1=Main feedback err · 2=Aux(reserved) · 4=Hall mismatch · 8=Current>peak ·
16=External inhibit · 32=Reserved · 64=Hall speed too high · **128=Speed tracking error**.
→ 오늘 서명 RED(MF=0x80, idle 발생)는 servo가 지령속도 추종 실패로 셧. 커뮤 미확립 상태로
MO=1 시 코히런트 모션이 안 나와 발생하는 것으로 해석(기어드 δ 재추첨과 정합).

**EAS 위저드 메커니즘 확정 (p.91, §16.3.10.4):** "Unit mode 3 is stepper mode... open-loop
mode that moves the electric field in internal stepping counts. The rotor follows the
electric field, allowing a motion with a constant torque (TC). Stepping resolution = 512
counts per pole pair (360°). **Stepper mode is mainly used for internal purposes during
wizard and phasing sequences.**"
→ 위저드/phasing = UM=3 스테퍼로 로터를 기지 전기각으로 당겨 절대엔코더 대비 오프셋 산출
(= 후보 B-2와 일치). 기어드 위저드 실패("Movements Uneven") = ±대칭 이동이 백래쉬에 깨져서.
→ **P0 §7 #6 부분 해소**: 위저드 프리미티브(UM=3 스테퍼 phasing) 확정. 상세 명령 시퀀스는
여전히 EAS 내부(문서 미기재)지만, 앱 자체 구현은 후보 B(CS/UM=3)로 충분히 재현 가능.
