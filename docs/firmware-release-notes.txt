Chapter 1. 
General  
The following Gold Level release notes include all modifications from version 01.01.07.15 released 
in January 2014 until version 01.02.18.00 B00 released in February 2024. 
Note  
Modifications between firmware versions may result in parameter database going back to default 
values. It is advisable to upload the drive’s parameters via the Elmo Application Studio (EAS) prior of 
updating any firmware version. The parameters can be downloaded to the drive after the firmware 
was downloaded.

2.1 Details 
Subject  
Details 
New Profile Position target 
processed in Quick stop 
From this version, new Set Points that the drive received via 
Object 0x607A are processed in the Quick Stop state as well. 
The outcome ofr the midification is that the motor will move to 
the new Set Point as soon as the state machine will be switched 
back to Operational Enable state. Previously, Set Points that 
were received during the Quick Stop state were ignored. 
New homing mode under Halt 
From this version, new DS402 Homing method that are 
requested when the drive is in Halt are processed and executed 
as soon as the “resume to motion” is permitted. Previously new 
Home methods were ignored.   
Tuba 30/480 High PWM 
switching 
From this version Tuba 30/480 can be activated in PWM 
frequency of 20KHz via the XP[2] command. Note that with this 
frequency the max current is limited to 20 amps. 
2.2 Bug fixes 1.2.18.00 B00 
Bug 
Details 
IAI sensor Zero Cross 
detection when position is 
inverted 
Zero Cross detection emulates Index allowing the application to 
e.g. perform Homing on Index or count a single turn event. 
With the IAI sensor, when the Zero Cross was used for Homing 
on Index, the capture position (Home position) was wrongly 
calculated when the sensor readout was defined as “Invert” via 
CA[28]. This was fixed. A correct capture is reported regardless 
of the “Invert bit” state.

3.1 Details 
Subject  
Details 
Present Current Command 
before cogging 
The Gold drive supports method to overcome a non-linear 
cogging disturbances. The current command before the cogging 
compensation is now reported via WI[49]. The value of the 
current command can also be mapped to a PDO via object 
0x2203 (Application Object) by setting 0x2F41 to 10.   
Stepper commutation 
improvement 
The stepper commutation procedure can result in an undesired 
fast current modification. From this version the current can be 
gradually decreased per SC[16]. The parameters is in seconds 
and default is 0.01.  
3.2 Bug fixes 1.2.17.05 B00 
Bug 
Details 
Non-Dual-Use speed limit 
In some situations the speed for a Q drives was wrongly 
calculated resulted in lower speed than required. Typically 
happened when using a high resolution sensor for the 
commutation. This was fixed. In any case the electrical speed is 
limited to 599Hz.   
CST mode speed calculation 
In Cyclic Synchronous Torque mode under Halls Only the speed 
calculation was wrong resulting in a wrong speed limit 
protection. This was fixed.  
CANOpen NMT reset after 
motor fault 
In some rare situations setting NMT mode 0x81 (reset Node) 
following a motor faults that occurred during the servo enable 
procedure (via MO=1 or via DS402 Operation Enable state), 
resulted in damage to the power bridge. The relevant faults are 
“Speed too High” and “Halls mismatch”.  This was fixed.

4.2 Bug fixes 1.2.17.04 B08 
Bug 
Details 
Handling SDO size under 
EtherCAT mailbox read option 
If the EtherCAT host request expedite SDO that is longer than 
the permitted size, the drive copies the maximum permitted 
size. Previously in such situations the drive fetched a wrong 
memory allocation and failed. In some situations the failure 
resulted in a request for another firmware download.   
Non-Dual-Use drives – lower 
speed when different sensor 
resolution is used. 
  
If different sensors were used for a Non-Dual-Use drives (Q 
Drives) and the resolution of the commutation sensor differ 
from the resolution of the velocity sensor, the speed limit 
preventing the dual use was wrongly calculated. The new 
CA[124] parameter denote the factor between the velocity and 
the commutation sensors in order to allow the proper speed. 
The units of the new parameter are 1/10000 which means that 
10000 (default) is ‘1’ which means that the commutation 
sensor and the velocity sensor have the same resolution.

9.1 Communication Processor Adjustment 
Due to a global component shortage, the firmware version of the drive was adjusted to fit a 
reduced memory chip-set.  
The modifications are directly related to the communication processor only and have no effect on 
the motion, servo, or other operational features of the drive. 
The new communication processor revision supports memory of 256K while the previous supported 
512K.  
The modification includes both boot and firmware versions. 
The modified version is relevant to all firmware files allowing the firmware downloading via all 
communication channels - FoE & USB\RS223. 
The modified version can be downloaded seamlessly to both types of drives - 256K or 512K.  
All present firmware versions support the 512K only. These versions cannot be downloaded to the 
256K drives. An attempt of doing so will result in an immediate error and the drive remains in boot 
state. 
In order to maintain compatibility with existing EtherCAT configuration files and minimize 
integration time, the product code (0x1018.2) and the revision number (0x1018.3) were not 
modified.

Telnet is no longer available in this version.

DHCP is no longer available, static addressing is required.

version via FoE 
The ability to update the boot loaded via FoE is no longer 
supported.

11.1

Subject

Twitter and Bee PWM 
multiplication enhanced   
PWM multiplication is done via XP[2] command. From this 
version the TWI & BEE with BV=100 & MC=20 can have 
multiplication of x3 (XP[2]=6).  
Note that the permitted current  limits will be reduced to 50% 
from the nominal one (when the multiplication is 1) 
Current phases sum tracking 
error 
 
The total current derived from the motor phases should be 0. 
From this version, ER[10] & ER[11] commands can be used to 
set threshold for differences of the current phases (ER[10]) with 
time window (ER[11]).  
The current sum tracking error assures that in situations that 
the sum of the current exceeds the threshold of ER[10] (in 
milliamps) for ER[11] (in milliseconds), than the motor is 
disabled with MF=0x4000000 and EMCY message 0x8382. 
Note: the parameter database will be reset to default. 
Velocity limit in Cyclic 
Synchronous Torque mode  
In CST mode the velocity can be limited to a desired value. This 
is done by reducing the current limit as follows: 
The velocity limit is set via object 0x6080 (in RPM). Via XA[10] 
the user sets the desired limits in 0.1% from 0x6080.  
For example: if the 0x6080 is 2000 [RPM] and the XA[10] is 800 
(i.e. 80%) than the torque will decrease from 1600 [RPM] which 
are the 80% from 0x6080 to 2000 [RPM] in such a way that in 
2000 RPM the torque will be 0.  
NF[9]: input shaping 
frequency limit improved 
The frequency of the input shaping are now improved for: 
NF[8]=1 (ZV mode) limit to 2.02Hz.  
NF[8]=2 (ZVD mode) limit to 4.05Hz.

When enabling the motor while there is a feedback error the EC

In these situations, the drive report EMCY 0x2311. 
Halls Encoder mismatch – 
CA[111]

Halls transition, hence any mismatch higher than

Recorder wrong state after LD

commands, the recorder status was wrongly reported as 
“waiting for trigger”. Status is reported via RR or SR command.

Enhanced stop of motion 
when drive in saturation 
When the drive is in voltage saturation, meaning that the bus 
voltage is so low, it prevents the motor from following the 
required speed command, and a speed tracking error is 
developed between the feedback and the speed command. 
When the motion is stopped (with the ST or Halt command) in

drive to stop the motor at the desired deceleration regardless 
of the tracking error caused by the voltage saturation.

This is done by using the two letters legacy method to 
determine the command offset starting from object 0x3000. 
Asymmetric current limits via 
0x2005  
Object 0x2005 can be used to address various functionalities of 
the drive.  
By setting GS[12]=3, the host can set asymmetric torque limits 
to the current controller. The low 16 bits are used for the low 
limits and the high 16 bits to the high limit. The limits affect 
the current command derived from the velocity controllers to 
the current controller.  
Units are 1/1000 of rated torque.
